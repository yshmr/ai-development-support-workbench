# Codex Task 01: LLMアプリ開発PoCのMVP実装

## 目的

このリポジトリに、要件メモから仕様整理・受け入れ条件・Jiraチケット・レビュー観点を生成するLLMアプリのMVPを実装してください。

## 前提

- 既存リポジトリがある場合は、まず構成を調査してください。
- 既存の技術スタックを優先してください。
- 新規構成が必要な場合は、Next.js + React + TypeScriptを前提にしてください。
- 破壊的な変更は避けてください。
- 大きな変更をする前に、実装方針を簡潔に説明してください。

## 実装したい機能

### 1. 入力フォーム

ユーザーが要件メモを入力できる画面を作ってください。

入力例:

```text
ユーザーがプロフィール画像を変更できるようにしたい。
画像は5MBまで、jpg/png対応。
変更後は即時反映したい。
失敗時にはエラーメッセージを表示したい。
```

### 2. 生成API

`POST /api/generate` を実装してください。

入力:

```json
{
  "inputText": "要件メモ本文"
}
```

出力:

```json
{
  "summary": "要件の要約",
  "spec": ["仕様項目1", "仕様項目2"],
  "acceptanceCriteria": ["受け入れ条件1", "受け入れ条件2"],
  "jiraTasks": [
    {
      "title": "チケットタイトル",
      "description": "チケット説明",
      "type": "frontend | backend | test | documentation"
    }
  ],
  "implementationPlan": ["実装方針1", "実装方針2"],
  "reviewPoints": ["レビュー観点1", "レビュー観点2"],
  "risks": ["リスク・確認事項1", "リスク・確認事項2"]
}
```

### 3. 構造化出力

LLMの出力は、できるだけ安定したJSON形式になるようにしてください。

- TypeScriptの型定義を作る
- 可能であればZodでバリデーションする
- LLM APIキーが未設定の場合でも、モック出力で画面確認できるようにする

### 4. 生成結果表示

生成された以下の項目を画面に見やすく表示してください。

- 要約
- 仕様
- 受け入れ条件
- Jiraチケット
- 実装方針
- レビュー観点
- リスク・確認事項

### 5. 生成履歴

MVPでは簡易的でよいので、生成履歴を保存してください。

優先順位:

1. ローカルJSON保存
2. SQLite + Prisma
3. ブラウザlocalStorage

既存構成に合うものを選んでください。

## テスト

最低限、以下を確認できるようにしてください。

- 空入力時にエラーになる
- APIキー未設定でもモック出力で動作確認できる
- 生成結果が期待する型に合っている
- 画面から入力して結果表示まで確認できる

READMEまたはdocsに、以下を追記してください。

- アプリを起動できる
- 入力フォームから要件メモを送信できる
- 生成結果が画面に表示される
- APIキー未設定でもモックで動作する
- READMEまたはdocsに使い方が記載されている
