# LLMアプリ開発PoC 追加サンプル入力案

保存済み履歴の評価対象はプロフィール画像変更の1種類のみだったため、今後の評価サンプルとして以下の2件を追加候補にする。

## サンプル入力案 1: 商品一覧の価格フィルタ

```text
商品一覧画面に価格帯フィルタを追加したい。
最低価格と最高価格を入力できる。
検索条件はURLに反映して、ページを再読み込みしても維持したい。
最低価格が最高価格を上回る場合は検索せず、エラーメッセージを表示したい。
```

期待する評価観点:

- 最低価格・最高価格の入力仕様が含まれる
- URLクエリへの反映が含まれる
- 再読み込み後の状態復元が含まれる
- 不正な価格範囲のバリデーションが含まれる
- UI、検索ロジック、テストのタスクに分解される

## サンプル入力案 2: 通知設定

```text
ユーザーがメール通知のオンオフを設定できるようにしたい。
通知種別はキャンペーン、セキュリティ、メンテナンスの3種類。
セキュリティ通知はオフにできない。
設定変更後は保存完了メッセージを表示したい。
保存に失敗した場合は元の設定に戻したい。
```

期待する評価観点:

- 3種類の通知種別が仕様に含まれる
- セキュリティ通知をオフにできない制約が含まれる
- 保存完了メッセージが受け入れ条件に含まれる
- 保存失敗時のロールバックまたは状態復元が含まれる
- バリデーションと権限制御のレビュー観点が含まれる
